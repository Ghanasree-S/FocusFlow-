🎯 Project Objective

Primary Goal: Apply and compare multiple time series forecasting techniques — LSTM (deep learning), ARIMA (statistical), Prophet (additive), and VAR (multivariate) — to predict daily productivity patterns from automatically tracked computer usage data.

Specific Objectives

  1. Time Series Data Collection: Build an automatic activity tracker that monitors active windows every 10 seconds, producing a continuous daily time series of productive minutes
  2. Univariate Forecasting: Implement and compare three time series models (LSTM, ARIMA, Prophet) for 7-day ahead productivity prediction
  3. Multivariate Analysis (VAR): Use Vector AutoRegression to model the bidirectional relationship between mood and productivity time series, testing Granger causality
  4. Adaptive Ensemble: Dynamically combine LSTM, ARIMA, and Prophet predictions using inverse-error weighted averaging that adapts per user over time
  5. Model Evaluation: Compare all models using MAE, RMSE, and MAPE metrics with proper train/test splits
  6. Stationarity Testing: Apply Augmented Dickey-Fuller (ADF) test and differencing to ensure data suitability for ARIMA modeling
  7. Visualization & Dashboard: Present forecasting results, model comparisons, and insights in an interactive web application

Why This Project?

  • Demonstrates end-to-end time series pipeline from raw data collection to multi-step forecasting
  • Practical comparison of 3 fundamentally different forecasting approaches: deep learning (LSTM), classical statistical (ARIMA), and decomposition-based (Prophet)
  • Explores multivariate time series via VAR and Granger causality testing
  • Shows how adaptive ensemble methods can outperform individual models by adjusting weights based on historical prediction errors
  • Real-world time series data with weekly seasonality, trends, and noise that challenges each model differently
  • Full-stack implementation with React + Flask + MongoDB for real-time forecasting application

📈 Time Series Forecasting Models

We implement and compare three fundamentally different time series forecasting approaches. Each captures different temporal patterns in the productivity data. Their predictions are combined via an adaptive weighted ensemble for superior forecasting accuracy.

1. LSTM (Long Short-Term Memory)

Deep Learning Neural Network Weight: 40%

What is LSTM?

A type of Recurrent Neural Network (RNN) designed to remember long-term dependencies. It uses "gates" to control information flow, avoiding the vanishing gradient problem.

Why LSTM for this project?

  • Captures complex non-linear patterns humans can't see
  • Learns sequential dependencies (if Monday productive, Tuesday might be too)
  • Handles variable length input (uses 7-day lookback window)

Architecture Used

model = Sequential([ LSTM(50, return_sequences=True, input_shape=(7, 1)), # 50 units, 7-day sequence Dropout(0.2), # Prevent overfitting LSTM(50), # Second LSTM layer Dropout(0.2), Dense(1) # Output: next day prediction ])

2. ARIMA (AutoRegressive Integrated Moving Average)

Statistical Classical ML Weight: 30%

What is ARIMA?

A classic statistical model used since the 1970s for time series forecasting. It combines autoregression (AR), differencing (I), and moving average (MA) components.

Our Configuration: ARIMA(1, 0, 0)

  • p = 1 (AR term): Use yesterday's value to predict today
  • d = 0 (Differencing): Data is already stationary, no trend removal needed
  • q = 0 (MA term): Don't use past prediction errors

The Formula

Today's Productivity = c + (φ × Yesterday's Productivity) + ε # Example calculation: # If yesterday = 200 min, coefficient φ = 0.8, constant c = 30 # Today ≈ 30 + (0.8 × 200) = 190 minutes

Why ARIMA for this project?

  • Simple and interpretable - easy to explain
  • Great for smooth trends - captures gradual changes
  • Lowest absolute error (MAE) in our evaluation

3. Prophet (by Meta/Facebook)

Seasonality Additive Model Weight: 30%

What is Prophet?

A forecasting tool developed by Facebook specifically for business time series with strong seasonal patterns and multiple seasonalities.

The Formula

y(t) = g(t) + s(t) + h(t) + ε(t) Where: g(t) = Trend (overall growth/decline) s(t) = Seasonality (weekly, monthly patterns) h(t) = Holiday effects ε(t) = Error term # Example for a Friday: # Base trend: 200 min # Weekly pattern: -30 min (Fridays are less productive) # Friday prediction: 170 min

Why Prophet for this project?

  • Automatic seasonality detection (weekday vs weekend patterns)
  • Handles missing data gracefully
  • Provides uncertainty intervals for predictions

Why Ensemble (Combine All Three)?

Each model has unique strengths and weaknesses. By combining them with weighted averaging, we get the "wisdom of the crowd" effect - reducing individual model errors!

# Ensemble prediction formula final_prediction = (0.4 × LSTM) + (0.3 × ARIMA) + (0.3 × Prophet) # Example: LSTM_pred = 180 min ARIMA_pred = 200 min Prophet_pred = 170 min ensemble = (0.4 × 180) + (0.3 × 200) + (0.3 × 170) = 72 + 60 + 51 = 183 min # Final prediction
Model Weight Best At Weakness
LSTM 40% Complex patterns, non-linear relationships Needs more data, slower to train
ARIMA 30% Smooth trends, lowest MAE Struggles with sudden changes
Prophet 30% Weekly seasonality, interpretable May overfit to seasonal patterns

📊 VAR Model & Granger Causality Analysis

Vector AutoRegression (VAR) extends univariate time series to the multivariate case. We use it to model the bidirectional temporal relationship between mood and productivity -- answering: Does mood drive productivity, or does productivity drive mood?

4. VAR (Vector AutoRegression)

Multivariate Granger Causality Impulse Response Bidirectional

What is VAR?

VAR is a multivariate time series model where each variable is modeled as a linear function of its own past values AND the past values of other variables. Unlike ARIMA (univariate), VAR captures cross-variable dependencies.

The VAR(p) Formula

# VAR(p) model for 2 variables (mood & productivity): Y(t) = c + A1*Y(t-1) + A2*Y(t-2) + ... + Ap*Y(t-p) + e(t) Where: Y(t) = [mood(t), productivity(t)] # 2-dimensional vector c = intercept vector Ai = 2x2 coefficient matrices # capture cross-dependencies p = lag order (selected by AIC) e(t) = error vector # Expanded for mood: mood(t) = c1 + a11*mood(t-1) + a12*productivity(t-1) + e1(t) # ^^ This coefficient shows how past # productivity affects today's mood

Granger Causality Test

We test causality in both directions using the Granger framework:

  • H0: Past values of X do NOT help predict Y (no Granger causality)
  • H1: Past values of X DO help predict Y (Granger causality exists)
  • If p-value < 0.05 for mood -> productivity: mood Granger-causes productivity
  • If p-value < 0.05 for productivity -> mood: productivity Granger-causes mood
  • Both can be true simultaneously (bidirectional causality)

Analysis Pipeline

  1. Data Alignment: Match daily mood entries with daily productivity metrics on matching dates
  2. Stationarity Check: ADF test on both series; apply differencing if non-stationary
  3. Lag Selection: Use AIC/BIC to determine optimal lag order p
  4. Pearson Correlation: Overall linear correlation between mood and productivity
  5. Cross-Correlation: Lagged correlations to determine which variable leads
  6. Granger Causality F-tests: Statistical tests in both directions with p-values
  7. VAR Forecasting: Joint 3-day forecasts of both mood and productivity
  8. Impulse Response Functions (IRF): How a one-unit "shock" to mood propagates to productivity over time

Implementation (statsmodels)

from statsmodels.tsa.api import VAR from statsmodels.tsa.stattools import grangercausalitytests, adfuller # Step 1: Stationarity check adf_mood = adfuller(data['mood']) adf_prod = adfuller(data['productivity']) # Step 2: Fit VAR model (lag order selected by AIC) model = VAR(data[['mood', 'productivity']]) fitted = model.fit(maxlags=3, ic='aic') # Step 3: Granger causality test granger = grangercausalitytests( data[['productivity', 'mood']], maxlag=3 ) # Step 4: Impulse Response Function (5-step ahead) irf = fitted.irf(periods=5) # Step 5: Forecast next 3 days forecast = fitted.forecast(data.values[-fitted.k_ar:], steps=3)

🔄 Adaptive Ensemble Weight Optimization

Instead of using fixed weights (0.4 LSTM + 0.3 ARIMA + 0.3 Prophet), we implement dynamically-optimized weights per user that automatically adjust based on each model's recent prediction accuracy. The ensemble learns which model works best for each individual user's time series pattern.

Adaptive Weighted Ensemble Algorithm

Per-User Inverse-Error Exponential Smoothing

Why Adaptive Weights?

User Pattern Best Model Reason
Highly regular schedule Prophet (higher weight) Strong weekly seasonality
Sudden habit change LSTM (higher weight) Better at learning new patterns
Stable trend, low noise ARIMA (higher weight) AR(1) captures persistence

Optimization Formula

# Step 1: Compute absolute errors for each model errors = { 'lstm': |actual - lstm_predicted|, 'arima': |actual - arima_predicted|, 'prophet': |actual - prophet_predicted| } # Step 2: Inverse-error weighting (lower error = higher weight) epsilon = 1e-6 inv_errors = {m: 1/(e + epsilon) for m, e in errors.items()} total = sum(inv_errors.values()) w_computed = {m: ie/total for m, ie in inv_errors.items()} # Step 3: Exponential smoothing (alpha=0.3) w_new = alpha * w_computed + (1 - alpha) * w_old # Example after a week of strong seasonality: # LSTM: 0.25 ARIMA: 0.28 Prophet: 0.47

Algorithm Properties

  • Initial weights: Equal (0.33, 0.33, 0.33) for new users
  • Smoothing factor alpha = 0.3: Balances responsiveness vs. stability
  • Weight bounds: Minimum 0.1 per model to prevent total exclusion
  • Update frequency: After each prediction cycle with known actuals
  • Convergence: Weights stabilize after ~10 prediction cycles

🔍 Key Findings & Learnings

Time Series Model Findings

  1. ARIMA(1,1,1) is Optimal: Simple AR(1) with first-order differencing outperformed complex configurations. Productivity is largely an autoregressive process — today's output is strongly influenced by yesterday's
  2. LSTM Needs Larger Datasets: With only 91 data points, LSTM couldn't fully leverage its deep learning capability. Time series with <200 observations favor statistical models
  3. Prophet Excels at Seasonality: Successfully captured that weekends have 30-40% lower productivity. The additive decomposition (trend + weekly seasonality) fits the data well
  4. Ensemble Reduces Prediction Variance: Combining models via adaptive weighting smooths out individual model prediction spikes and reduces overall forecast uncertainty
  5. VAR Requires Sufficient Observations: Granger causality tests need at least 7 days of aligned mood+productivity data to produce statistically meaningful results
  6. ADF Test Confirms Non-Stationarity: Raw productivity time series required first-order differencing before ARIMA modeling. The differenced series passed the ADF test at 5% significance level

Time Series Patterns Discovered

  • Weekly Seasonality: Clear 7-day periodicity — weekday productivity is 35% higher than weekends
  • Autoregressive Nature: Strong lag-1 autocorrelation (today's productivity predicts tomorrow's)
  • Peak Hours: 9 AM - 11 AM shows highest productive time (intra-day pattern)
  • Post-Lunch Dip: 1 PM - 3 PM shows increased distraction (circadian effect on time series)
  • Monday Surge: Start of week shows 5% higher productivity than weekly average
  • Mood-Productivity Correlation: Cross-correlation analysis shows mood leads productivity by ~1 day

Challenges Faced & Solutions

Challenge Solution
Non-stationary time series for ARIMA Applied ADF test + first differencing (d=1) to achieve stationarity
LSTM sequence length tuning Experimented with lookback windows of 7, 14, 30 days; 7-day window optimal
Prophet overfitting to holidays Disabled holiday effects; kept only weekly seasonality component
Ensemble weight instability Exponential smoothing (alpha=0.3) + minimum weight bounds (0.1)
VAR mood data sparsity Minimum 7-day threshold before running analysis; heuristic fallback otherwise
TensorFlow import errors Implemented fallback mode using statistical predictions when LSTM unavailable
Model retraining overhead Auto-retrain every 20 new activities using checkpoint system

📝 Project Summary

Time Series Forecasting Summary

ChronosAI is a time series forecasting application that predicts daily productivity patterns using four complementary approaches: LSTM (deep learning with gated recurrent units for sequential pattern learning), ARIMA (classical statistical autoregressive model with differencing for stationarity), Prophet (additive decomposition capturing weekly seasonality and trends), and VAR (multivariate model analyzing mood-productivity bidirectional Granger causality). An adaptive ensemble dynamically combines LSTM, ARIMA, and Prophet forecasts using inverse-error weighted averaging with exponential smoothing. The system automatically collects time series data from computer usage, applies stationarity tests (ADF), and produces 7-day ahead forecasts that are compared across models using MAE, RMSE, and MAPE metrics.
